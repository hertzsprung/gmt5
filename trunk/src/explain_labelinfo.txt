BD(+a)IT(angle)
For annotations at a fixed angle, BD(+an) for line-normal, or BD(+ap) for line-parallel [Default].
#ifdef CONTOUR
By appending the BD(u) or BD(d) we get annotations whose top face the next upper or lower annotation, respectively.
#endif
.TP
BD(+c)IT(dx)[/IT(dy)]
Sets the clearance between label and optional text box.  Append BD(c)|BD(i)|BD(p) to specify
the unit or % to indicate a percentage of the label font size [15%].
.TP
BD(+d)
Turns on debug which will draw helper points and lines to illustrate the workings of the quoted line setup.
.TP
BD(+e)
Delay the plotting of the text.  This is used to build a clip path based on the text, then lay down other
overlays while that clip path is in effect, then turning of clipping with psclip OPT(Ct)  which finally
plots the original text.
.TP
BD(+f)IT(font)
Sets the desired font [Default BD(FONT_ANNOT_PRIMARY) with its size changed to 9p].
(See SPECIFYING FONTS below).
.TP
BD(+g)[IT(color)]
Selects opaque text boxes [Default is transparent]; optionally specify the color [Default is BD(PS_PAGE_COLOR)].
(See SPECIFYING COLOR below).
.TP
BD(+j)IT(just)
Sets label justification [Default is MC].  Ignored when OPT(SqN)|BD(n)+|-1 is used.
.TP

BD(+Lh)
Take the label from the current segment header (first scan for an embedded
OPT(L)IT(label) option, if not use the first word following the segment flag).
For multiple-word labels, enclose entire label in double quotes.
.TP
BD(+Ld)
Take the Cartesian plot distances along the line as the label; append BD(c)|BD(i)|BD(p)
as the unit [Default is BD(PROJ_LENGTH_UNIT)].
.TP
BD(+LD)
Calculate actual map distances; append BD(d|e|f|k|n|M|n|s) as the unit [Default is BD(d)(egrees),
unless label placement was based on map distances along the lines in which case we use the same unit specified for that algorithm].
Requires a map projection to be used.
.TP
BD(+Lf)
Use text after the 2nd column in the fixed label location file as the label.  Requires the fixed label
location setting.
.TP
BD(+Lx)
As BD(+Lh) but use the headers in the IT(xfile.d) instead.  Requires the crossing file option.

BD(+n)IT(dx)[/IT(dy)]
Nudges the placement of labels by the specified amount (append BD(c)|BD(i)|BD(p) to specify the units).
Increments are considered in the coordinate system defined by the orientation of the line; use
BD(+N) to force increments in the plot x/y coordinates system [no nudging].
.TP
BD(+o)
Selects rounded rectangular text box [Default is rectangular].  Not applicable for curved text (BD(+v)) and
only makes sense for opaque text boxes.
.TP
BD(+p)[IT(pen)]
Draws the outline of text boxes [Default is no outline]; optionally specify pen for outline [Default is width = 0.25p,
color = black, style = solid].
(See SPECIFYING PENS below).
.TP

Will not place labels where the line's radius of curvature is less than \fImin_rad\fP [Default is 0].\"'

BD(+u)IT(unit)
Appends IT(unit) to all line labels. If IT(unit) starts with a leading hyphen (-) then there will be no space
between label value and the unit.
#ifdef GRID
If no IT(unit) is appended we use the units listed in the grid file.
#endif
[Default is no unit].
.TP
BD(+v)
Specifies curved labels following the path [Default is straight labels].
.TP
BD(+w)
Specifies how many (IT(x),IT(y)) points will be used to estimate label angles [Default is 10].
.TP
BD(+=)IT(prefix)
Prepends IT(prefix) to all line labels.
If IT(prefix) starts with a leading hyphen (-) then there will be no space
between label value and the prefix. [Default is no prefix].
