GMT Bug Notes

To file a bug report, please send e-mail to GMT Help. Note that due to the ever-growing presence of spam, you will need to subscribe to this mailing-list in order to post. We apologize if this may be a hassle to you, but it saves the 1000+ subscribers from seeing all the junk spam otherwise would get through. Make sure your email includes the following information:
  • Version of GMT you are using.
  • Hardware platform and Operating System, including version.
  • Individual GMT program that may have a bug.
  • A clear explanation of the problem.
  • Example script or command line and, if necessary, test data that recreate the problem. If the problem occurs for a huge data set, try to see if the problem can be reproduced with a smaller subset. Do not attach data files that exceed one Mb - we will contact you should we need further information.
  • Other pieces of relevant information
Note that sometimes a work-around can be crafted while waiting for the next release. You may want to check the SOEST GMT Wiki to see if anything relevant have been added.
