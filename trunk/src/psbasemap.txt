.SH NAME
psbasemap \- To plot \fIPostScript\fP basemaps
.SH SYNOPSIS
\fBpsbasemap\fP GMT_B_OPT GMT_J_OPT GMT_Rgeoz_OPT 
[ GMT_E_OPT ] [ OPT(G)\fIfill\fP ] [ GMT_Jz_OPT ] [ GMT_K_OPT ] 
[ GMT_SCALE ] [ GMT_O_OPT ] [ GMT_P_OPT ] [ GMT_U_OPT ] 
[ GMT_TROSE ] [ GMT_V_OPT ] [ GMT_X_OPT ] [ GMT_Y_OPT ] [ OPT(Z)\fIzlevel\fP ] [ GMT_c_OPT ]
.SH DESCRIPTION
\fBpsbasemap\fP creates \fIPostScript\fP code that will produce a basemap.  Several map projections
are available, and the user may specify separate tickmark intervals for boundary annotation,
ticking, and [optionally] gridlines.  A simple map scale or directional rose may also be plotted.

.SH EXAMPLES
The following section illustrates the use of the options by giving some examples for the
available map projections.  Note how scales may be given in several different ways
depending on the projection.  Also note the use of upper case letters to specify map width
instead of map scale.
.SH NON-GEOGRAPHICAL PROJECTIONS
.SS Linear x-y plot
To make a linear x/y frame with all axes, but with only left and bottom axes annotated, using xscale = yscale = 1.0,
ticking every 1 unit and annotating every 2, and using xlabel = "Distance" and ylabel = "No of samples", use
.br
.sp
\fBpsbasemap\fP OPT(R)0/9/0/5 OPT(Jx)1 OPT(Bf)1\fBa\fP2:Distance:/:"No of samples":\fBWeSn\fP > linear.ps
.SS Log-log plot
To make a log-log frame with only the left and bottom axes, where the x-axis is LEN_1 cm and annotated
every 1-2-5 and the y-axis is LEN_2 cm and annotated every power of 10 but has tickmarks every 0.1, run
.br
.sp
\fBpsbasemap\fP OPT(R)1/10000/1e20/1e25 SCALE_LL OPT(B)2:Wavelength:/a1\fBpf\fP3:Power:\fBWS\fP > loglog.ps
.SS Power axes
To design an axis system to be used for a depth\-sqrt(age) plot with depth positive down, ticked and
annotated every 500m, and ages annotated at 1 my, 4 my, 9 my etc, use
.br
.sp
\fBpsbasemap\fP OPT(R)0/100/0/5000 OPT(Jx)1\fBp\fP0.5/-0.001 OPT(B)1\fBp\fP:"Crustal age":/500:Depth: > power.ps
.SS Polar (theta,r) plot
For a base map for use with polar coordinates, where the radius from 0 to 1000 should correspond
to LEN_P inch and with gridlines and ticks every 30 degrees and 100 units, use
.br
.sp
\fBpsbasemap\fP OPT(R)0/360/0/1000 SCALE_P OPT(B)30\fBp\fP/100 > polar.ps
.SH CYLINDRICAL MAP PROJECTIONS
.SS Cassini
A LEN_C-cm-wide basemap using the Cassini projection may be obtained by
.br
.sp
\fBpsbasemap\fP OPT(R)20/50/20/35 SCALE_C OPT(P) OPR(B)5\fBg\fP5:.Cassini: > cassini.ps
.SS Mercator [conformal]
A Mercator map with scale LEN_M inch/degree along equator, and showing the length of 5000 km along the equator
(centered on POS_M inch), may be plotted as
.br
.sp
\fBpsbasemap\fP OPT(R)90/180/-50/50 SCALE_M OPT(B)30\fBg\fP30:.Mercator: MAP_M > mercator.ps
.SS Miller
A global Miller cylindrical map with scale 1:200,000,000 may be plotted as
.br
.sp
\fBpsbasemap\fP OPT(Rg) OPT(Jj)180/1:200000000 OPT(B)30\fBg\fP30:.Miller: > miller.ps
.SS Oblique Mercator [conformal]
To create a page-size global oblique Mercator basemap for a pole at (90,30) with gridlines every 30 degrees, run
.br
.sp
\fBpsbasemap\fP  OPT(R)0/360/-70/70 SCALE_O OPT(B)30\fBg\fP30:."Oblique Mercator": > oblmerc.ps
.SS Transverse Mercator [conformal]
A regular Transverse Mercator basemap for some region may look like
.br
.sp
\fBpsbasemap\fP OPT(R)69:30/71:45/-17/-15:15 OPT(Jt)70/1:1000000 OPT(B)15\fBm\fP:."Survey area": OPT(P) > transmerc.ps
.SS Equidistant Cylindrical Projection
This projection only needs the central meridian and scale.  A LEN_Q cm wide global basemap centered on the 130E
meridian is made by
.br
.sp
\fBpsbasemap\fP OPT(R)-50/310/-90/90 SCALE_Q OPT(B)30\fBg\fP30:."Equidistant Cylindrical": > cyl_eqdist.ps
.SS Universal Transverse Mercator [conformal]
To use this projection you must know the UTM zone number, which defines the central meridian.  A UTM
basemap for Indo-China can be plotted as
.br
.sp
\fBpsbasemap\fP OPT(R)95/5/108/20\fBr \-Ju\fP46/1:10000000 OPT(B)3\fBg\fP3:.UTM: > utm.ps
.SS Cylindrical Equal-Area
First select which of the cylindrical equal-area projections you want by deciding on the standard parallel.
Here we will use 45 degrees which gives the Gall-Peters projection.  A LEN_Y inch wide global basemap centered on the Pacific
is made by
.br
.sp
\fBpsbasemap\fP OPT(Rg) SCALE_Y OPT(B)30\fBg\fP30:.Gall-Peters: > gall-peters.ps
.SH CONIC MAP PROJECTIONS
.SS Albers [equal-area]
A basemap for middle Europe may be created by
.br
.sp
\fBpsbasemap\fP OPT(R)0/90/25/55 SCALE_B OPT(B)10\fBg\fP10:."Albers Equal-area": > albers.ps
.SS Lambert [conformal]
Another basemap for middle Europe may be created by
.br
.sp
\fBpsbasemap\fP OPT(R)0/90/25/55 SCALE_L OPT(B)10\fBg\fP10:."Lambert Conformal Conic": > lambertc.ps
.SS Equidistant
Yet another basemap of width LEN_D inch for middle Europe may be created by
.br
.sp
\fBpsbasemap\fP OPT(R)0/90/25/55 SCALE_D OPT(B)10\fBg\fP10:."Equidistant conic": > econic.ps
.SH AZIMUTHAL MAP PROJECTIONS
.SS Lambert [equal-area]
A LEN_A-cm-wide global view of the world from the vantage point -80/-30 will give the following basemap:
.br
.sp
\fBpsbasemap\fP OPT(Rg) SCALE_A OPT(B)30\fBg\fP30/15\fBg\fP15:."Lambert Azimuthal": > lamberta.ps
.br
.sp
Follow the instructions for stereographic projection if you want to impose rectangular
boundaries on the azimuthal equal-area map but substitute OPT(Ja) for OPT(Js).
.SS Equidistant
A LEN_E-cm-wide global map in which distances from the center (here 125/10) to any point is true can be obtained by:
.br
.sp
\fBpsbasemap\fP OPT(Rg) SCALE_E OPT(B)30\fBg\fP30/15\fBg\fP15:."Equidistant": > equi.ps
.SS Gnomonic
A view of the world from the vantage point -100/40 out to a horizon of 60 degrees from the center can be made using
the Gnomonic projection:
.br
.sp
\fBpsbasemap\fP OPT(Rg) SCALE_F OPT(B)30\fBg\fP30/15\fBg\fP15:."Gnomonic": > gnomonic.ps
.SS Orthographic
A global perspective (from infinite distance) view of the world from the vantage point 125/10 will give
the following LEN_G-inch-wide basemap:
.br
.sp
\fBpsbasemap\fP OPT(Rg) SCALE_G1 OPT(B)30\fBg\fP30/15\fBg\fP15:."Orthographic": > ortho.ps
.SS General Perspective
The OPT(JG) option can be used in a more generalized form, specifying altitude above the surface,
width and height of the view point, and twist and tilt.
A view from 160 km above -74/41.5 with a tilt of 55 and azimuth of 210 degrees, and limiting the
viewpoint to 30 degrees width and height will product a LEN_G-inch-wide basemap:
.br
.sp
\fBpsbasemap\fP OPT(Rg) SCALE_G2 OPT(B)5\fBg\fP1/5\fBg\fP1:."General Perspective": > genper.ps
.SS Stereographic [conformal]
To make a polar stereographic projection basemap with radius = LEN_S cm to \-60 degree latitude, with plot title
"Salinity measurements", using 5 degrees annotation/tick interval and 1 degree gridlines, run
.br
.sp
\fBpsbasemap\fP OPT(R)-45/45/-90/-60 SCALE_S1 OPT(B)5\fBg\fP5:."Salinity measurements": > stereo1.ps
.br
.sp
To make a LEN_S-cm-wide stereographic basemap for Australia from an arbitrary view point (not the poles), and use a
rectangular boundary, we must give the pole for the new projection and use the OPT(R) option to
indicate the lower left and upper right corners (in lon/lat) that will define our rectangle.  We
choose a pole at 130/-30 and use 100/-45 and 160/-5 as our corners.  The command becomes
.br
.sp
\fBpsbasemap\fP OPT(R)100/-45/160/-5\fBr SCALE_S2 OPT(B)30\fBg\fP30/15\fBg\fP15:."General Stereographic View": > stereo2.ps
.SH MISCELLANEOUS MAP PROJECTIONS
.SS Hammer [equal-area]
The Hammer projection is mostly used for global maps and thus the spherical form is used.
To get a world map centered on Greenwich at a scale of 1:200000000, use
.br
.sp
\fBpsbasemap\fP OPT(Rg) OPT(Jh)180/1:200000000 OPT(B)30\fBg\fP30/15\fBg\fP15:.Hammer: > hammer.ps
.SS Sinusoidal [equal-area]
To make a sinusoidal world map centered on Greenwich, with a scale along the equator of LEN_I inch/degree, use
.br
.sp
\fBpsbasemap\fP OPT(Rd) SCALE_I OPT(B)30\fBg\fP30/15\fBg\fP15:."Sinusoidal": > sinus1.ps
.br
.sp
To make an interrupted sinusoidal world map with breaks at 160W, 20W, and 60E, with a scale along the equator of 0.02 inch/degree, run the following sequence of commands:
.br
.sp
\fBpsbasemap\fP OPT(R)-160/-20/-90/90 OPT(Ji)-90/0.02\fBi\fP OPT(B)30\fBg\fP30/15\fBg\fP15\fBWesn\fP OPT(K) > sinus_i.ps
.br
\fBpsbasemap\fP OPT(R)-20/60/-90/90 OPT(Ji)20/0.02\fBi\fP OPT(B)30\fBg\fP30/15\fBg\fP15\fBwesn\fP OPT(O) OPT(K) OPT(X)2.8\fBi\fP >> sinus_i.ps
.br
\fBpsbasemap\fP OPT(R)60/200/-90/90 OPT(Ji)130/0.02\fBi\fP OPT(B)30\fBg\fP30/15\fBg\fP15\fBwEsn\fP OPT(O) OPT(X)1.6\fBi\fP >> sinus_i.ps
.br
.SS Eckert IV [equal-area]
Pseudo-cylindrical projection typically used for global maps only.  Set the central longitude and scale, e.g.,
.br
.sp
\fBpsbasemap\fP OPT(Rg) SCALE_K4 OPT(B)30\fBg\fP30/15\fBg\fP15:."Eckert IV": > eckert4.ps
.SS Eckert VI [equal-area]
Another pseudo-cylindrical projection typically used for global maps only.  Set the central longitude and scale, e.g.,
.br
.sp
\fBpsbasemap\fP OPT(Rg) SCALE_K6 OPT(B)30\fBg\fP30/15\fBg\fP15:."Eckert VI": > eckert6.ps
.SS Robinson
Projection designed to make global maps "look right".  Set the central longitude and width, e.g.,
.br
.sp
\fBpsbasemap\fP OPT(Rd) SCALE_N OPT(B)30\fBg\fP30/15\fBg\fP15:."Robinson": > robinson.ps
.SS Winkel Tripel
Yet another projection typically used for global maps only.  You can set the central longitude, e.g.,
.br
.sp
\fBpsbasemap\fP OPT(R)90/450/-90/90 SCALE_R OPT(B)30\fBg\fP30/15\fBg\fP15:."Winkel Tripel": > winkel.ps
.SS Mollweide [equal-area]
The Mollweide projection is also mostly used for global maps and thus the spherical form is used.
To get a LEN_W-cm-wide world map centered on the Dateline:
.br
.sp
\fBpsbasemap\fP OPT(Rg) SCALE_W OPT(B)30\fBg\fP30/15\fBg\fP15:.Mollweide: > mollweide.ps
.SS Van der Grinten
The Van der Grinten projection is also mostly used for global maps and thus the spherical form is used.
To get a LEN_V-inch-wide world map centered on the Dateline:
.br
.sp
\fBpsbasemap\fP OPT(Rg) SCALE_V OPT(B)30\fBg\fP30/15\fBg\fP15:."Van der Grinten": > grinten.ps
.SH RESTRICTIONS
For some projections, a spherical earth is implicitly assumed.  A warning will notify the user if OPT(V)
is set. Also note that plot titles are not plotted if OPT(E) is given.
.SH BUGS
The OPT(B) option is somewhat complicated to explain and comprehend.  However, it is fairly simple for
most applications (see examples).
