.TP
GMT_J_OPT
Selects the map projection. The following character determines the projection. If the
character is upper case then the argument(s) supplied as scale(s) is interpreted to be
the map width (or axis lengths), else the scale argument(s) is the map scale (see its
definition for each projection). UNIT is cm, inch, or point, depending on the BD(PROJ_LENGTH_UNIT)
setting in BD(gmt.conf), but this can be overridden on the command line by appending
BD(c), BD(i), or BD(p) to the IT(scale) or IT(width) values.  Append BD(h), BD(+), or BD(-)
to the given IT(width) if you instead want to set map height, the maximum dimension, or
the minimum dimension, respectively [Default is BD(w) for width].
.br
In case the central meridian is an optional parameter and it is being omitted, then the
center of the longitude range given by the OPT(R) option is used. The default standard parallel
is the equator.
.br
The ellipsoid used in the map projections is user-definable by editing the BD(gmt.conf) file
in your home directory. 73 commonly used ellipsoids and spheroids are currently
supported, and users may also specify their own custum ellipsoid parameters [Default is WGS-84].
Several GMT parameters can affect the projection: BD(PROJ_ELLIPSOID), BD(GMT_INTERPOLANT),
BD(PROJ_SCALE_FACTOR), and BD(PROJ_LENGTH_UNIT); see the BD(gmt.conf) man page for details.
.br

BD(AZIMUTHAL PROJECTIONS:)
.sp
Except for polar aspects, OPT(R)w/e/s/n will be reset to OPT(Rg).  Use OPT(R)<...>BD(r) for smaller regions.
.TP
OPT(Ja)IT(lon0/lat0)[IT(/horizon)]IT(/scale) or OPT(JA)IT(lon0/lat0)[IT(/horizon)]IT(/width) (Lambert BD([E])).
IT(lon0/lat0) specifies the projection center.
IT(horizon) specifies the max distance from projection center (in degrees, <= 180, default 90).
Give IT(scale) as BD(1:)IT(xxxx) or IT(radius/lat), where IT(radius) is distance
in UNIT from origin to the oblique latitude IT(lat).

OPT(Jg)IT(lon0/lat0)[IT(/horizon)]IT(/scale) or OPT(JG)IT(lon0/lat0)[IT(/horizon)]IT(/width) (Orthographic).
IT(lon0/lat0) specifies the projection center.
IT(horizon) specifies the max distance from projection center (in degrees, <= 90, default 90).

.TP
OPT(Jg)IT(lon0/lat0/altitude/azimuth/tilt/twist/Width/Height/scale) or OPT(JG)IT(lon0/lat0/altitude/azimuth/tilt/twist/Width/Height/width) (General Perspective).
IT(lon0/lat0) specifies the projection center.
IT(altitude) is the height (in km) of the viewpoint above local sea level.
If IT(altitude) is less than 10, then it is the distance from the center of the earth
to the viewpoint in earth radii. If IT(altitude) has a suffix BD(r) then it is the radius
from the center of the earth in kilometers.
IT(azimuth) is measured to the east of north of view.
IT(tilt) is the upward tilt of the plane of projection. If IT(tilt) is negative, then the
viewpoint is centered on the horizon.
Further, specify the clockwise IT(twist), IT(Width), and
IT(Height) of the viewpoint in degrees.
Give IT(scale) as BD(1:)IT(xxxx) or IT(radius/lat), where IT(radius) is distance
in UNIT from origin to the oblique latitude IT(lat).
.TP
OPT(Js)IT(lon0/lat0)[IT(/horizon)]IT(/scale) or OPT(JS)IT(lon0/lat0)[IT(/horizon)]IT(/width) (General Stereographic BD([C])).
IT(lon0/lat0) specifies the projection center.
IT(horizon) specifies the max distance from projection center (in degrees, < 180, default 90).
Give IT(scale) as BD(1:)IT(xxxx) (true at pole) or IT(lat0)/BD(1:)IT(xxxx) (true at standard parallel IT(lat0))
or IT(radius/lat) (IT(radius) in UNIT from origin to the oblique latitude IT(lat)).
Note if BD(1:)IT(xxxx) is used then to specify IT(horizon) you must also specify the IT(lat0) as +-90 to avoid ambiguity.

BD(NON-GEOGRAPHICAL PROJECTIONS:)
.TP
OPT(Jp)[BD(a)]IT(scale)[IT(/origin)][BD(r)|BD(z)] or OPT(JP)[BD(a)]IT(width)[IT(/origin)][BD(r)|BD(z)] (Polar coordinates (theta,r))
Optionally insert BD(a) after OPT(Jp) [ or OPT(JP)] for
azimuths CW from North instead of directions CCW from East [Default].
Optionally append /IT(origin) in degrees to indicate an angular offset [0]).
Finally, append BD(r) if r is elevations in degrees (requires s >= 0 and n <= 90)
or BD(z) if you want to annotate depth rather than radius [Default].
Give IT(scale) in UNIT/r-unit.
.TP
OPT(Jx)IT(x-scale)[IT(/y-scale)] or OPT(JX)IT(width)[IT(/height)] (Linear, log, and power scaling)
Give IT(x-scale) (BD(1:)IT(xxxx) or UNIT/x-unit) and/or IT(y-scale) (BD(1:)IT(xxxx) or UNIT/y-unit); or
specify IT(width) and/or IT(height) in UNIT.
IT(y-scale)=IT(x-scale) if not specified separately and using BD(1:)IT(xxxx) implies that x-unit and y-unit are in
meters.
Use negative scale(s) to reverse the direction of an axis (e.g., to have y be positive down). Set
IT(height) or IT(width) to 0 to have it recomputed based on the implied scale of the other axis.
Optionally, append to IT(x-scale), IT(y-scale), IT(width) or IT(height) one of the following:
.RS
.TP
.B d

.TP
.B l

.TP
BD(p)IT(power)
Raise values to IT(power) before scaling.
.TP

.TP

.PP
Default axis lengths (see BD(gmt.conf)) can be invoked
using OPT(JXh) (for landscape); OPT(JXv) (for portrait) will swap the x- and y-axis lengths.
The default unit for this installation is either cm or inch, as defined in the file
BD(share/gmt.conf). However, you may change this by editing your BD(gmt.conf) file(s).
