.TH GMTAVERAGE 1 MANDATE GMT_STRING "Generic Mapping Tools"
.SH NAME
gmtaverage \- filter to block average (IT(x),IT(y),IT(z)) data.
.SH SYNOPSIS
BD(gmtaverage) [ IT(xyz[w]file(s)) ] GMT_I_OPT GMT_Rx_OPT OPT(Te)|BD(m)|BD(n)|BD(o)|BD(s)|BD(w)|IT(quantile) 
[ OPT(C) ] [ OPT(E)[BD(b)] ] [ OPT(Q) ] 
[ GMT_V_OPT ] [ OPT(W)[BD(io)] ] [ GMT_b_OPT ] [ GMT_f_OPT ] [ GMT_h_OPT ] 
[ GMT_i_OPT ] [ GMT_o_OPT ] [ GMT_r_OPT ] [ GMT_colon_OPT ]
.SH DESCRIPTION
BD(gmtaverage) reads arbitrarily located (IT(x),IT(y),IT(z)) triples [or optionally weighted
quadruples (IT(x),IT(y),IT(z),IT(w))] from standard input [or IT(xyz[w]file(s))] and writes to
standard output an average position and value for every non-empty block in a grid region
defined by the OPT(R) and OPT(I) arguments.  BD(gmtaverage) should be used as a pre-processor
before running BD(surface) to avoid aliasing short wavelengths, but is also generally useful for
decimating or averaging (IT(x),IT(y),IT(z)) data.  You can modify the precision of the output
format by editing the BD(FORMAT_FLOAT_OUT) parameter in your BD(gmt.conf) file, or
you may choose binary input and/or output using single or double precision storage.
.SH REQUIRED ARGUMENTS
.TP
IT(xyz[w]file(s))
3 [or 4] column ASCII file(s) [or binary, see OPT(b)] holding (IT(x),IT(y),IT(z)[,IT(w)])
data values.  [IT(w)] is an optional weight for the data.  If no file is specified,
BD(gmtaverage) will read from standard input.
#include "explain_-I.txt"
#include "explain_-R.txt"
.TP
OPT(T)
Specify the operator to use when computing output values.  Choose among BD(e) (median), BD(m) (mean),
BD(n) (number of points), BD(o) (mode), BD(s) (sum), BD(w) (weight sum), or the IT(quantile) of the distribution to be returned.
Here, 0 < IT(quantile) < 1.
.SH OPTIONAL ARGUMENTS
.TP
OPT(C)
Use the center of the block as the output location [Default uses the mean, median, or mode x and y
as location, depending on operator selected with OPT(T) (but see OPT(Q))].
.TP
OPT(E)[BD(b)]
Provide Extended report which includes BD(s) (the scale of the reported value),
BD(l), the lowest value, and BD(h), the high value for each block. Output order
becomes IT(x),IT(y),IT(z),IT(s),IT(l),IT(h)[,IT(w)]. [Default outputs IT(x),IT(y),IT(z)[,IT(w)].
For box-and-whisker calculation using OPT(Te), select OPT(Eb) which will output IT(x),IT(y),IT(z),IT(l),IT(q25),IT(q75),IT(h)[,IT(w)],
where IT(q25) and IT(q75) are the 25% and 75% quantiles, respectively.
The IT(scale) is the standard deviation, the L1 scale, or the LMS scale, depending on OPT(T).
See OPT(W) for IT(w) output.
.TP
OPT(Q)
(Quicker) Finds median (or mode) IT(z) and (IT(x),IT(y)) at that median (or mode) IT(z) [Default finds median or mode
IT(x) and IT(y) independent of IT(z)].  Also see OPT(C).  Ignored if OPT(Tm), OPT(Tn), OPT(Ts) or OPT(Tw) is used.
#include "explain_-V.txt"
.TP
OPT(W)[BD(io)]
Weighted modifier[s].  Unweighted input and output has 3 columns IT(x),IT(y),IT(z); Weighted
i/o has 4 columns IT(x),IT(y),IT(z),IT(w).  Weights can be used in input to construct weighted
output values in blocks.  Weight sums can be reported in output for later combining several
runs, etc.  Use OPT(W) for weighted i/o, OPT(Wi) for weighted input only, OPT(Wo) for
weighted output only.  [Default uses unweighted i/o].

Each block is the
locus of points nearest the grid value location.  For example, with OPT(R)10/15/10/15
and and OPT(I)1:  with the OPT(r) option 10 <= (IT(x),IT(y)) < 11 is one of 25 blocks;
without it 9.5 <= (IT(x),IT(y)) < 10.5 is one of 36 blocks.

.SH EXAMPLES
To find 5 by 5 minute block mode values from the double precision binary data in hawaii_b.xyg
and output an ASCII table, run
.sp
BD(gmtaverage) hawaii_b.xyg OPT(R)198/208/18/25 OPT(I)5BD(m) OPT(To) OPT(bi)3 > hawaii_5x5.xyg
.br
.sp
To find 5 by 5 minute block mean values from the same file, run
.sp
BD(gmtaverage) hawaii.xyg OPT(R)198/208/18/25 OPT(I)5BD(m) OPT(Tm) > hawaii_5x5.xyg
.br
.sp
To find the number of data points in each 5 by 5 minute block from the same file, run
.sp
BD(gmtaverage) hawaii.xyg OPT(R)198/208/18/25 OPT(I)5BD(m) OPT(Tn) > hawaii_5x5.xyn
.br
.sp
To compute the shape of a data distribution per bin via a box-and-whisker diagram we need the 0%, 25%,
50%, 75%, and 100% quantiles.  To do so on a global 5 by 5 degree basis from the ASCII table depths.xyz
and send output to an ASCII table, run
.sp
BD(gmtaverage) depths.xyz OPT(Rg) OPT(I)5 OPT(Te) OPT(Eb) OPT(r) > depths_5x5.txt
