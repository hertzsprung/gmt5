.TH GRDSAMPLE 1 MANDATE GMT_STRING "Generic Mapping Tools"
.SH NAME
grdsample \- Resample a grid onto a new lattice
.SH SYNOPSIS
BD(grdsample) IT(in_grdfile) OPT(G)IT(out_grdfile) [ GMT_I_OPT ] 
[ GMT_Rgeo_OPT ] [ OPT(T) ] [ GMT_V_OPT ] 
[ GMT_fi_OPT ] [ GMT_n_OPT ] [ GMT_r_OPT ]
.SH DESCRIPTION
BD(grdsample) reads a grid file and interpolates it to create a new grid file with either:  a different registration 
(OPT(r) or OPT(T)); or, a new grid-spacing or number of nodes (OPT(I)), and perhaps also a new sub-region 
(OPT(R)).  A bicubic [Default], bilinear, B-spline or nearest-neighbor interpolation is used; see OPT(n) for settings.
Note that using OPT(R) only is equivalent to BD(grdcut) or BD(grdedit \-S).  BD(grdsample) safely 
creates a fine mesh from a coarse one; the converse may suffer aliasing unless the data are filtered using
BD(grdfft) or BD(grdfilter).
.br
When OPT(R) is omitted, the output grid will cover the same region as the input grid.
When OPT(I) is omitted, the grid spacing of the output grid will be the same as the input grid.
Either OPT(r) or OPT(T) can be used to change the grid registration. When omitted, the output grid will
have the same registration as the input grid.

.SH HINTS
If an interpolation point is not on a node of the input grid, then a NaN at any node in the neighborhood 
surrounding the point will yield an interpolated NaN.  Bicubic interpolation [default] yields continuous 
first derivatives but requires a neighborhood of 4 nodes by 4 nodes.  Bilinear interpolation [OPT(n)]
uses only a 2 by 2 neighborhood, but yields only zero-order continuity.  Use bicubic when smoothness is 
important.  Use bilinear to minimize the propagation of NaNs.
.SH EXAMPLES
To resample the 5 x 5 minute grid in hawaii_5by5_topo.nc onto a 1 minute grid:
.br
.sp
grdsample hawaii_5by5_topo.nc -I1m -Ghawaii_1by1_topo.nc
.br
.sp
To translate the gridline-registered file surface.nc to pixel registration while
keeping the same region and grid interval:
.br
.sp
grdsample surface.nc -T -Gpixel.nc
