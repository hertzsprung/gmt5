.SH NAME
grdvector \- Plot vector field from two component grids
.SH SYNOPSIS
BD(grdvector) IT(compx.nc) IT(compy.nc) GMT_J_OPT [ OPT(A) ] 
[ GMT_B_OPT ] [ OPT(C)IT(cptfile) ] 
[ OPT(G)IT(fill) ] [ GMT_I_OPT ] [ GMT_K_OPT ] [ OPT(N) ] [ GMT_O_OPT ] [ GMT_P_OPT ] 
[ OPT(Q)IT(parameters) ] [ GMT_Rgeo_OPT ] [ OPT(S)[BD(l)]IT(scale) ] 
[ OPT(T) ] [ GMT_U_OPT ] [ GMT_V_OPT ] [ OPT(W)IT(pen) ] [ GMT_X_OPT ] 
[ GMT_Y_OPT ] [ OPT(Z) ] [ GMT_c_OPT ] [ GMT_f_OPT ] [ GMT_p_OPT ] [ GMT_t_OPT ]
.SH DESCRIPTION
BD(grdvector) reads two 2-D grid files which represents the x- and y-components of a vector field and
produces a vector field plot by 
drawing vectors with orientation and length according to the information in the files.  Alternatively,
polar coordinate components may be used (r, theta).
BD(grdvector) is basically a short-hand for using 2 calls to BD(grd2xyz) and pasting the output through
BD(psxy) OPT(SV).
#include "explain_commonitems.txt"
.SH REQUIRED ARGUMENTS
.TP
IT(compx.nc)
Contains the x-component of the vector field.
.TP
IT(compy.nc)
Contains the y-component of the vector field. 
(See GRID FILE FORMATS below.)
#include "explain_-J.txt"
.SH OPTIONAL ARGUMENTS
.TP
OPT(A)
Means grid files have polar (r, theta) components instead of Cartesian (x, y).
#include "explain_-B.txt"
.TP
OPT(C)IT(cptfile)
Use IT(cptfile) to assign colors based on vector length.
.TP
OPT(G)IT(fill)
Sets color or shade for vector interiors [Default is no fill].
.TP
OPT(I)
Only plot vectors at nodes every IT(x_inc), IT(y_inc) apart (must be multiples of
original grid spacing).  Append BD(m) for arc minutes or BD(s) for arc seconds. [Default plots every node].
#include "explain_-K.txt"
.TP
OPT(N)
Do NOT clip vectors at map boundaries [Default will clip].
#include "explain_-O.txt"
#include "explain_-P.txt"
.TP
OPT(Q)IT(parameters)
Modify vector parameters.  For vector heads, append vector head IT(size) [Default is 0, i.e., stick-plot].  
Several modifiers may be appended to specify placement of vector heads, their shapes, and the justification of the vector:
.br
MOD(a)IT(angle) sets the angle of the vector head apex [30].
.br
MOD(b) places a vector head at the beginning of the vector path [none].
.br
MOD(e) places a vector head at the end of the vector path [none].
.br
MOD(l) draws half-arrows, using only the left side [both].
.br
MOD(r) draws half-arrows, using only the right side [both].
.br
MOD(n)IT(norm) scales down vector attributes (pen thickness, head size) with decreasing length,
where vectors shorter than IT(norm) will have their
attributes scaled by length/IT(norm) [arrow attributes remains invariant to length].
.br
.sp
MOD(j)IT(just) determines how the input IT(x),IT(y) point relates
to the vector.  Choose from BD(b)eginning [default], BD(e)nd, or BD(c)enter.
#include "explain_-R.txt"
Specify a subset of the grid.
.TP
OPT(S)[BD(l)]IT(scale)
Sets scale for vector length in data units per distance measurement unit [1].  Append BD(c), BD(i), or BD(p)
to indicate the measurement unit (cm, inch,or point).  Prepend BD(l) to indicate a fixed length for
all vectors.
.TP
OPT(T)
Means azimuth should be converted to angles based on the selected map projection.
#include "explain_-U.txt"
#include "explain_-V.txt"
.TP
OPT(W)IT(pen)
Set pen attributes used for vector outlines [Default: width = default, color = black, style = solid].
#include "explain_-XY.txt"
.TP
OPT(Z)
Means the angles provided are azimuths rather than direction (requires OPT(A)).

.SH EXAMPLES
To draw the vector field given by the files r.nc and theta.nc on a linear plot
with scale LEN1 cm per data unit,
using vector rather than stick plot, scale vector magnitudes so that LEN2 units
equal 1 inch, and center vectors on the node locations, run 
.br
.sp
grdvector r.nc theta.nc SCALE -A -Q0.1i+e+jc SCL2 > gradient.ps
.br
.sp
