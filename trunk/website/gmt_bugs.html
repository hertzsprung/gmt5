GMT Bug Notes

To file a bug report, please send e-mail to GMT Help. Note that due to the ever-growing presence of spam, you will need to subscribe to this mailing-list in order to post. We apologize if this may be a hassle to you, but it saves the 1000+ subscribers from seeing all the junk spam that otherwise would get through. Make sure your email includes the following information:
  • Version of GMT you are using (e.g., type psxy - to see what is reported).
  • Is this a 32-bit or 64-bit installation of GMT?
  • If GMT 5, did you install using the GMT 4 compatibility mode?
  • Hardware platform and Operating System, including version (e.g., under Unix-like systems, type uname -a to get details; for Windows, just tell us what version).
  • Individual GMT program that may have a bug.
  • A clear explanation of the problem.
  • Example script or command line and, if necessary, test data that recreate the problem. If the problem occurs for a huge data set, try to see if the problem can be reproduced with a smaller subset. Do not attach data files that exceed one Mb - we will contact you should we need further information.
  • Other pieces of relevant information
The easier we can reproduce the problem then the quicker we can fix it. Some bugs are platform-dependent and can only be reproduced on that system; we have several Linux-flavors running in a simulator for that purpose. The CVS version of GMT will immediately be updated once a fix has been found. Note that sometimes a work-around can be crafted while waiting for the next official release. You may want to check the SOEST GMT Wiki to see if anything relevant have been added.
