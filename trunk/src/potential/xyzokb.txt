.TH GRDCUT 1 MANDATE GMT_STRING "Generic Mapping Tools"
.SH NAME
grdcut \- Extract a subregion out of a grid file
.SH SYNOPSIS
BD(grdcut) IT(input_file.nc) OPT(G)IT(output_file.nc) GMT_Rgeo_OPT 
[ GMT_V_OPT ] [ OPT(Z)[BD(n)]IT(min/max)] ] [ GMT_f_OPT ]
.SH DESCRIPTION
BD(grdcut) will produce a new IT(output_file.nc) file which is a subregion of IT(input_file.nc).
The subregion is specified with OPT(R) as in other programs; the
specified range must not exceed the range of IT(input_file.nc).  If in doubt, run BD(grdinfo)
to check range.  Alternatively, define the subregion indirectly via a range check on the node values.
Complementary to BD(grdcut) there is BD(grdpaste), which will join together two
grid files along a common edge.
.TP
IT(input_file.nc)
this is the input IT(.nc) format file.
.TP
OPT(G)IT(output_file.nc)
this is the output IT(.nc) format file.
#include "explain_-R.txt"
This defines the subregion to be cut out.
.SH OPTIONS
#include "explain_-V.txt"
.TP
OPT(Z)
Determine the new rectangular region so that all nodes outside this region are also outside the given IT(z)-range [-inf/+inf].
To indicate no limit on min or max, specify a hyphen (-).  Normally, any NaNs encountered are simply skipped.  Use
OPT(Zn) to consider a NaN to be outside the IT(z)-range.
#include "explain_-f.txt"
#include "explain_help.txt"
#include "explain_grd_inout.txt"

.SH EXAMPLES
Suppose you have used BD(surface) to grid ship gravity in the region between 148E - 162E and 8N - 32N, and you 
do not trust the gridding near the edges, so you want to keep only the area between 150E - 160E and 10N - 30N, then: 
.sp
BD(grdcut) grav_148_162_8_32.nc OPT(G)grav_150_160_10_30.nc OPT(R)150/160/10/30 OPT(V)
To return the subregion of a grid such that any boundary strips where all values are entirely
above 0, try 
.sp
BD(grdcut) bathy.nc OPT(G)trimmed_bathy.nc OPT(Z)-/0 OPT(V)
