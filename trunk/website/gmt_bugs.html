GMT Bug Notes

17-OCT-2000: GMT Version 3.3.6 Released

This page contains links to pieces of GMT source that have been updated in response to bug reports, and in some cases due to minor enhancements. No documentation of such enhancements will be available until the next release. Click on the links below to receive the updated file in question.

Libraries and include files

  • gmt_io.c
    1. 12/22/2000. We revert our policy and allow -: to work on any kind of data, just as pre-3.3.4.
  • gmt_support.c
    1. 12/22/2000. Use Mika Heiskanen's binary search for cpt index, which speeds up grdview/image for large cpt files.
    2. 12/08/2000. When D_FORMAT was not %lg, we format to check if a unit string should be added.
    3. 11/28/2000. While preparing the boundary conditions, the test for xrange being a multiple of xinc could fail due to a bug in the test.
    4. 10/30/2000. GMT_getinc, which parses the -Idx/dy options for grid spacings, would temporarily remove any trailing m or c (for minute or seconds) and not put it back when done. Thus, the command string in output gridfiles would show a slighlty incorrect command line, e.g., it would say -I2 instead of -I2m. The grid interval was correctly set so this is just a cosmetic issue.
  • gmt_map.c
    1. 12/09/2000. Made grdproject give bogus 2nd message about search radius.
    2. 12/08/2000. Must add 360 if both west and east are negative.
    3. 12/05/2000. There should be no Gnomonic annotations except for rectangular regions.
    4. 11/05/2000. The -JXd projection is a special version of linear with built-in knowledge of the 360-degree periodicity. Unfortunately, the inverse projection was not properly implemented; this bug only seems to have affected the z-axis on 3-D basemaps.
    5. 10/20/2000. A _very_ densily sampled track crossing a map boundary could run into a GMT-roundoff situation where a crossing is first detected but later discarded - resulting in a missing "pen down" instruction (and hence a PostScript violation). The fix is to use the pre-round-off coordinates to decide whether to discard or not.
  • gmt.h
    1. 12/08/2000. Added a parameter needed for 3-D basemaps.
  • gmt_init.h
    1. 12/08/2000. Added a parameter needed for 3-D basemaps.
  • gmt_plot.c
    1. 12/08/2000. Minor changes to ensure the user could specify a custom D_FORMAT _and_ us -B, at the same time.
  • gmt_shore.c
    1. 12/03/2000. A problem arises in pscoast when -A is used with a minimum size larger than zero. The levels of the bin nodes are based on -A0 but that can change if another cutoff is used. This occasionally results in bad behavior. The fix improves this situation alot but it is not a complete fix.
  • gmt_io.c
    1. 10/20/2000. Parsing of -Z did check for errors but did not notify users why it failed - thus grd2xyz would produce an empty output file and no error messages. This is more of a 'documentation' issue than a real bug, though.

    Individual programs

  • minmax.c
    1. 12/15/2000. Added option -L for geographical data.
  • psxy.c
    1. 12/09/2000. -SE will work for points that are outside map but whose ellipse may clip the map.
  • psxyz.c
    1. 12/09/2000. -SE will work for points that are outside map but whose ellipse may clip the map.
  • psscale.c
    1. 12/08/2000. Now, -B0 will plot no anotations at all.
  • pswiggle.c
    1. 12/07/2000. Positive anomaly was not always painted in the preferred direction.
    2. 11/17/2000. The -Z option assumed values were given in inches unless a modifier was appended. However, it should instead check what the default measure unit is set to (MEASURE_UNIT).
  • mapproject.c
    1. 12/05/2000. Wrong units sometimes used when reporting xmin/xmax/ymin/ymax; however the output coordinates were always correct.
  • pscoast.c
    1. 12/05/2000. Had world_map = TRUE for Gnomonic, which is not correct, and this could lead to criss-crossing lines across the map.
  • grdvector.c
    1. 11/30/2000. Added -Sl option to specify a fixed length for all vectors.
  • mex/grdinfo2.c
    1. 11/17/2000. If the file was not found, the exit return would shut down Matlab.
  • mex/grdread.c
    1. 11/17/2000. If the file was not found, the exit return would shut down Matlab.
  • mgg/gmttrack.c
    1. 11/03/2000. The -Sa -Sb options did not convert km to meters so km was compared to meters internally. Wrong loop variable used to initialize distance (this only affected runs with -Sa, -Sb).
  • img2grd
    1. 10/30/2000. Would fail if -V were not given.
    These bug fixes will be incorporated into the next release.
